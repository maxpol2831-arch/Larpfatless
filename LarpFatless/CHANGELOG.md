# Changelog

## 2026-07-05

- Доработана PWA v2: добавлен обязательный онбординг профиля с расчётом нормы по формуле Миффлина-Сан Жеора.
- Добавлено главное меню с разделами Профиль, ИИ-чат, Дневник и AI Calories.
- Раздел AI Calories получил выбор фото, превью перед анализом, результат с предупреждением о примерном расчёте и сохранение в дневник.
- Добавлены лёгкие пасхалки: долгий тап по аватару, ночное приветствие, реакция ИИ-чата на шуточную фразу и баннер серии по калориям.
- Backend `/api/analyze` теперь отдаёт понятную ошибку при лимитах Gemini `429` / `RESOURCE_EXHAUSTED`.
- Добавлена веб/PWA-версия `apps/web` на React + Vite для деплоя на Vercel.
- Добавлен serverless endpoint `apps/web/api/analyze.ts` для безопасного вызова Gemini через `GEMINI_API_KEY`.
- Добавлены PWA-манифест, service worker, иконка приложения и install-flow через `beforeinstallprompt`.
- Добавлены текстовый, голосовой и фото-ввод для PWA, включая сжатие изображения через canvas.
- Добавлено локальное хранение PWA-дневника и профиля через IndexedDB.
- Добавлена инструкция деплоя: `docs/pwa-deploy-vercel.md`.
- Усилен парсинг результата ИИ-анализа фото: поддержка markdown-обертки, смешанного текста и алиасов полей.
- Улучшена обработка ошибок анализа фото на mobile: таймаут, понятные сообщения, аккуратная передача MIME-типа.
- Добавлено логирование диагностики ответов модели на backend.
- Добавлены авторизация по email/паролю, серверное хранение профиля и синхронизация приемов пищи.
- Добавлены редактирование и удаление приемов пищи, а также snackbar с отменой удаления.
- Добавлено локальное сохранение состояния приложения через AsyncStorage.
- Исправлена синхронизация дневника: пустой серверный список теперь корректно заменяет локальный кэш.
- Переделана система темизации: светлая, темная и системная темы теперь применяются через единый `ThemeProvider`.
- Переключатель темы в профиле теперь работает на всем приложении и сохраняется между запусками.
