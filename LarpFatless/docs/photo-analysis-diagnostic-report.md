# Диагностика бага распознавания еды по фото

Дата: 2026-07-05

## Что было найдено

1. Ответ ИИ-модели приходил не всегда в "чистом" JSON.
Иногда ответ содержал markdown-обертку, поясняющий текст до или после JSON и альтернативные имена полей (`kcal`, `protein`, `fat`, `carbs`, `weight`).

2. Клиент и сервер были чувствительны к сетевым сбоям.
При долгом ответе модели пользователь мог зависать в состоянии загрузки без понятного сообщения.

3. Передача фото с мобильного клиента требовала более аккуратного определения MIME-типа.
Это особенно важно для iOS, где URI и тип файла могут отличаться от ожидаемого сценария.

## Что исправлено

1. Добавлен устойчивый парсер ответа модели:
- очистка markdown fences;
- извлечение JSON из смешанного текста;
- нормализация альтернативных имен полей к единой схеме приложения.

2. Улучшена диагностика на backend:
- добавлен preview сырого ответа модели в non-production режиме;
- добавлено логирование неуспешных попыток анализа.

3. Улучшена обработка ошибок на mobile:
- таймаут запроса 15 секунд через `AbortController`;
- более понятные тексты ошибок для пользователя;
- корректное определение MIME-типа изображения по расширению файла.

## Ожидаемый результат после фикса

- Фото из камеры и галереи корректно отправляются на backend.
- Даже если модель вернула JSON в markdown-обертке или с иными названиями полей, результат корректно отображается в приложении.
- При сетевой ошибке или таймауте пользователь получает понятное сообщение и может повторить попытку или перейти к ручному вводу.
