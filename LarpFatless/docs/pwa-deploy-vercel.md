# Деплой PWA LarpFatless на Vercel

Веб-версия приложения находится в `apps/web`. Это отдельное React + Vite PWA-приложение с serverless endpoint `/api/analyze`.

## Локальная проверка

Из корня проекта:

```powershell
.\start-web.cmd
```

Открыть:

```text
http://localhost:4173
```

Production-сборка:

```powershell
.\build-web.cmd
```

Готовые файлы появляются в `apps/web/dist`.

## Настройка Vercel

1. Создайте проект на Vercel и подключите репозиторий `LarpFatless`.
2. В настройках проекта укажите:
   - Root Directory: `apps/web`
   - Framework Preset: `Vite`
   - Build Command: `pnpm build`
   - Output Directory: `dist`
   - Install Command: `pnpm install`
3. В разделе Environment Variables добавьте:
   - `GEMINI_API_KEY` - ключ Google Gemini API
   - `GEMINI_MODEL` - `gemini-2.5-flash` или другая совместимая multimodal-модель
4. Нажмите Deploy.

## Важно по безопасности

`GEMINI_API_KEY` используется только в `apps/web/api/analyze.ts`. Он не попадает в клиентский JavaScript bundle и не должен добавляться в `.env` с префиксом `VITE_`.

## Проверка после деплоя

- Откройте сайт по HTTPS-ссылке Vercel.
- При первом запуске заполните профиль. До сохранения профиля главное меню не открывается.
- В главном меню проверьте переходы: `Профиль`, `ИИ-чат`, `Дневник`, `AI Calories`.
- В `ИИ-чат` отправьте описание еды, например `гречка 200 г и куриная грудка 150 г`, проверьте результат и сохраните его в дневник.
- В `AI Calories` выберите фото блюда, убедитесь, что появилось превью, затем нажмите `Анализировать`.
- В `Дневник` проверьте запись за сегодня, недельную сводку и удаление записи.
- Проверьте установку PWA через кнопку в шапке или меню браузера.
- Отключите интернет и убедитесь, что дневник открывается из IndexedDB, а анализ показывает offline-баннер.
- Для пасхалки зажмите аватар в шапке примерно на секунду или напишите в чат `я съел слона`.
